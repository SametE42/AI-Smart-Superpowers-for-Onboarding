# Roadmap

This roadmap describes realistic repository-improvement phases. It is not a release promise and does not claim external adoption or official integration by third-party tools.

## Phase 1: Framework Clarity

- Improve README positioning.
- Add comparison and research-oriented documentation.
- Document this repository's architecture.

## Phase 2: Template Expansion

- Add Context Index.
- Add Evidence Map.
- Add Risk Register.
- Add Freshness.
- Add Human Review Gates.
- Add Tech Stack.
- Add Build and Test.
- Add Dependencies.
- Add Runtime Environment.

## Phase 3: Complete Multilingual Support

- Build a dynamic language inventory.
- Add File-Maps for all existing languages.
- Support Canonical Structure Mode.
- Support Localized Structure Mode.
- Add language-support matrix.
- Add localization glossary.
- Track translation review status.
- Validate localized output.

## Phase 4: Language-Dependent Output

- Add language selection.
- Add localized folders.
- Add localized filenames.
- Add localized template content.
- Add installer.
- Add Output Manifest.

## Phase 5: Technology And Stack Neutrality

- Add generic stack detection.
- Add stack-specific guidance.
- Add build and test documentation.
- Add dependency documentation.
- Add runtime documentation.

## Phase 6: Tool Compatibility

- Keep tool-entrypoint templates short.
- Add integration notes.
- Avoid official-support claims unless they are evidenced.

## Phase 7: Skills

- Add repo-onboarding skill.
- Add docs-review skill.
- Add docs-update skill.

## Phase 8: Validation

- Add Context-Budget checks.
- Add Evidence checks.
- Add Freshness checks.
- Add Tool-Entrypoint checks.
- Add Localized-Output checks.
- Add Language-Support checks.
- Add File-Map-Schema checks.
- Add Stack-Detection checks.

## Phase 9: Examples

- Add minimal examples.
- Add standard examples.
- Add enterprise examples.
- Add monorepo examples.
- Add before-after examples.
- Add security-sensitive examples.
- Add legacy-codebase examples.
- Add localized examples.
- Add stack-specific examples.

## Phase 10: Release Readiness

- Prepare v1.0.0 release-note goals.
- Update changelog.
- Refine issue templates.
- Polish documentation.
