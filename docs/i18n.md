# Internationalization and Translation Strategy

English is the primary language and source of truth for this repository.

The current AI documentation is organized by language directly under `ai/`:

```text
ai/
├─ English/      # source of truth
├─ German/       # localized translation
├─ Spanish/      # localized translation
├─ French/       # localized translation
└─ ...
```

Legacy files under `i18n/` may remain for compatibility, but new AI documentation should be added under `ai/<Language>/`.

## Language policy

| Language role | Rule |
|---|---|
| Primary language | English |
| Source of truth | `ai/English/` and root-level English documentation |
| Localized documentation | Mirrored under `ai/<Language>/` |
| Conflict handling | English source wins unless maintainers explicitly decide otherwise |

## Coverage and AI translation status

Structural coverage and translation status are tracked separately.

- A language is structurally covered when it mirrors every Markdown file from `ai/English/`.
- A non-English page is AI-translated when it carries the AI quality-pass marker.
- `scripts/validate_repository.py` reports missing AI translation markers and legacy review markers.
- Changing translation status markers is release-significant and should be documented in `ai/TRANSLATION_STATUS.md` and `CHANGELOG.md`.

## Translation rules

- Keep directory names, file names, commands, paths, API names, class names and configuration keys unchanged unless a file is specifically intended to be localized.
- Preserve safety, security and permission boundaries exactly.
- Do not add new normative rules only in translated files. Add them to the English source first.
- Mark AI-generated translations clearly with the required AI status marker.
- Keep localized files structurally aligned with the English source file.
- Keep every language-folder `README.md` aligned with the canonical onboarding entrypoint: evidence-first Pre-Development Onboarding layer, `docs/ai/` target output, Superpowers-style disclaimer, Quickstart, Use/Not-Use cases and source links.

## Required status marker

Every localized document should include this machine-readable marker near the top:

```html
<!-- translation-status: ai-translated; ai-quality-pass -->
```

It should also include a human-readable status note:

```text
> Translation status: AI-translated from the English source; structural quality gate passed; human linguistic review still required unless translation_review_status is reviewed.
> Source language: English
> Source file: ai/English/<path>
> If this translation conflicts with the English source, the English source is authoritative.
```

## Recommended workflow

1. Update the English source file.
2. Mirror the file path in each target language.
3. Translate the content while preserving paths, commands and safety rules.
4. Run link and structure validation.
5. Confirm that every non-English page carries the AI quality-pass marker.

Use the maintenance script for bulk refreshes:

```bash
python scripts/refresh_ai_manual.py --root . --languages French Spanish Turkish Arabic
```

Then regenerate validation reports:

```bash
python scripts/validate_repository.py --root . --json ai/VALIDATION_REPORT.json --markdown ai/VALIDATION_REPORT.md
```
