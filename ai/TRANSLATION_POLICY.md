# Translation Policy

English is the source of truth. Every language folder should mirror the English structure and keep the same filenames.

Language-folder `README.md` files must also mirror the canonical English entrypoint structure for the evidence-first Pre-Development Onboarding layer: Overview, Where This Fits, Target Output, Quickstart, Source Of Truth And Links, Workflow, When To Use, When Not To Use, Manual Structure, safety rules, localization notes and quality checklist.

## Rules

1. Do not delete English source files.
2. Keep paths identical across languages.
3. Keep commands, code blocks, filenames and API names unchanged.
4. Translate explanations, headings and instructions.
5. Keep invariant terms visible when they define repository behavior: `evidence-first`, `Pre-Development Onboarding layer`, `docs/ai/`, `Superpowers-style`, `Magical Prompt Improver` and `obra/superpowers`.
6. Do not add compatibility, endorsement or integration claims for Superpowers-style workflows.
7. Mark incomplete translations clearly.
8. Review technical translations before publishing.
