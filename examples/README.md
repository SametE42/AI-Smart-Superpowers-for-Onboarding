# Examples

This folder contains practical examples for applying the AI onboarding standard to different repository types and agent workflows.

## Current Examples

| Example | Shows |
|---|---|
| `before-after/` | How scattered assumptions become structured context. |
| `minimal/` | Smallest useful onboarding structure. |
| `standard/` | Normal application onboarding. |
| `enterprise/` | Governance, Risk Register and human-review gates. |
| `monorepo/` | Multiple packages or services with bounded context. |
| `security-sensitive/` | Stronger boundaries for secrets, auth and deployment. |
| `legacy-codebase/` | Unknowns, fragile behavior and undocumented decisions. |
| `localized-de/` | German localized output under `docs/ki/`. |
| `localized-multilingual/` | Multiple localized structures through File-Maps. |
| `stacks/` | Python, TypeScript, Java, C#, Go, Rust and Infrastructure-as-Code hints. |
